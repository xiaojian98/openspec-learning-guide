# 00 - 快速开始

> 5 分钟体验 OpenSpec 核心功能

## 前置条件

- Node.js 20.19.0 或更高版本
- npm / pnpm / yarn 任一包管理器

## 第一步：安装 OpenSpec

```bash
npm install -g @fission-ai/openspec@latest
```

验证安装：

```bash
openspec --version
```

## 第二步：创建测试项目

```bash
# 创建一个空项目
mkdir my-first-openspec
cd my-first-openspec

# 初始化 git（推荐）
git init
```

## 第三步：初始化 OpenSpec

```bash
openspec init
```

你会看到交互式提示：

```
? 选择你要使用的 AI 工具：
  ◉ Claude Code
  ◯ Cursor
  ◯ Windsurf
  ◯ GitHub Copilot
  ...
```

选择你使用的工具（比如 Claude Code），然后确认。

## 第四步：查看生成的文件

```bash
ls -la .claude/
```

你会看到类似这样的结构：

```
.claude/
├── commands/
│   └── openspec/
│       ├── propose.md
│       ├── apply.md
│       └── archive.md
└── skills/
    └── openspec-*/
        └── SKILL.md
```

这些是 AI 工具的"指令文件"，告诉 AI 如何使用 OpenSpec。

## 第五步：创建你的第一个变更

在 Claude Code 中输入：

```
/opsx:propose 添加 README 文件
```

AI 会：
1. 创建 `openspec/changes/add-readme/` 目录
2. 生成 `proposal.md`（为什么做、做什么）
3. 生成 `design.md`（怎么实现）
4. 生成 `tasks.md`（任务清单）

## 第六步：查看生成的变更

```bash
# 查看所有变更
openspec list

# 查看变更详情
openspec show add-readme
```

## 第七步：实现变更

```
/opsx:apply
```

AI 会按照 `tasks.md` 中的任务逐步实现。

## 第八步：归档变更

实现完成后：

```
/opsx:archive
```

变更会被移动到 `openspec/changes/archive/` 目录，并且相关规范会合并到主规范中。

## 🎉 恭喜！

你已经完成了 OpenSpec 的基本工作流：

```
propose → apply → archive
```
