# 12 - 模板系统

> 自定义模板让 AI 生成更符合你需求的内容

## 模板是什么？

模板是 Markdown 文件，指导 AI 如何生成 artifact 内容。

```
┌─────────────────────────────────────────────────────────────────┐
│                     模板工作原理                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   AI 收到的完整指令：                                           │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ <context>                                               │   │
│   │ 技术栈: TypeScript + React                              │   │
│   │ ...config.yaml 中的 context...                          │   │
│   │ </context>                                              │   │
│   │                                                         │   │
│   │ <rules>                                                 │   │
│   │ - 规则 1                                                │   │
│   │ - 规则 2                                                │   │
│   │ </rules>                                                │   │
│   │                                                         │   │
│   │ <instruction>                                           │   │
│   │ schema.yaml 中的 instruction 字段                       │   │
│   │ </instruction>                                          │   │
│   │                                                         │   │
│   │ <template>                                              │   │
│   │ 模板文件内容（templates/proposal.md）                   │   │
│   │ </template>                                             │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 模板语法

### 1. 章节标题

```markdown
## Why

<!-- Explain the motivation for this change -->

## What Changes

<!-- Describe what will change -->
```

AI 会根据标题和注释填充内容。

### 2. HTML 注释指导

```markdown
## Why

<!-- 说明为什么需要这个变更。关注问题，不是解决方案。 -->
```

注释是给 AI 的指导，不会出现在最终输出中。

### 3. 示例格式

```markdown
## Capabilities

### New Capabilities

<!-- 使用 kebab-case 命名，例如：user-auth, data-export -->

- `<name>`: <简要描述>
```

提供格式示例帮助 AI 理解期望的输出格式。

### 4. 条件章节

```markdown
## Migration Plan

<!-- 仅当涉及数据迁移时填写此章节 -->

## Open Questions

<!-- 如有未决问题，在此列出。没有则删除此章节 -->
```

## 自定义模板示例

### Proposal 模板（增强版）

```markdown
<!-- templates/proposal.md -->
## Why

<!-- 说明为什么需要这个变更：
     - 解决什么问题？
     - 对谁有价值？
     - 为什么现在做？ -->

## What Changes

<!-- 描述将要改变的内容：
     - 新增功能
     - 修改的功能
     - 删除的功能 -->

## Capabilities

### New Capabilities

<!-- 新增的能力。使用 kebab-case 命名 -->

- `<capability-name>`: <简要描述>

### Modified Capabilities

<!-- 修改的现有能力。说明什么在改变 -->

- `<existing-capability>`: <什么在改变>

### Removed Capabilities

<!-- 删除的能力。说明为什么删除 -->

## Impact

### 代码影响
<!-- 列出受影响的代码模块 -->

### API 影响
<!-- 列出受影响的 API -->

### 依赖影响
<!-- 列出新增或变更的依赖 -->

## Risks

<!-- 识别潜在风险 -->

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| <!-- 风险描述 --> | 高/中/低 | 高/中/低 | <!-- 如何缓解 --> |

## Success Criteria

<!-- 如何验证变更成功？ -->

- [ ] <!-- 成功标准 1 -->
- [ ] <!-- 成功标准 2 -->
```

### Tasks 模板（带时间估算）

```markdown
<!-- templates/tasks.md -->
## 1. <任务组名称>

<!-- 预计时间: X 小时 -->

- [ ] 1.1 <!-- 任务描述 --> (`预估: Xm`)
- [ ] 1.2 <!-- 任务描述 --> (`预估: Xm`)

## 2. <任务组名称>

<!-- 预计时间: X 小时 -->

- [ ] 2.1 <!-- 任务描述 --> (`预估: Xm`)
- [ ] 2.2 <!-- 任务描述 --> (`预估: Xm`)

---

## 总预计时间

<!-- 自动计算 -->

| 阶段 | 预计时间 |
|------|---------|
| 1. <任务组> | X 小时 |
| 2. <任务组> | X 小时 |
| **总计** | **X 小时** |
```

## 模板最佳实践

### ✅ 好的模板

```markdown
## Why

<!-- 1-2 段话说明动机。回答：什么问题？为什么现在解决？ -->

## Impact

<!-- 列出受影响的：代码、API、数据库、依赖 -->
```

**特点**：
- 简洁明确
- 给出指导但不过度约束
- 结构清晰

### ❌ 不好的模板

```markdown
## Why

你必须写以下内容：
1. 第一段写背景（50-100字）
2. 第二段写动机（100-150字）
3. 第三段写预期效果（50-80字）
4. 必须引用至少 3 个用户反馈
5. 必须包含竞品分析表格
...
```

**问题**：
- 过度约束，AI 难以发挥
- 格式过于死板
- 可能不适合所有场景

## 查看模板内容

```bash
# 查看某个 artifact 的完整指令（包含模板）
openspec instructions proposal --change my-change

# JSON 格式
openspec instructions proposal --change my-change --json | jq '.template'
```
