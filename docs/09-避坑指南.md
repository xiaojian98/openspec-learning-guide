# 09 - 避坑指南

> 常见错误 + 解决方案

## 入门常见问题

### 1. 忘记先初始化项目

❌ **错误**：直接运行 `/opsx:propose`

✅ **正确**：
```bash
openspec init
```

**原因**：没有初始化就没有 `.claude/` 目录，AI 不知道如何使用 OpenSpec。

---

### 2. 在规范中写实现细节

❌ **错误**：
```markdown
### Requirement: 用户登录
系统必须使用 bcrypt 加密密码，存储在 users 表中。
```

✅ **正确**：
```markdown
### Requirement: 用户登录
系统必须验证用户身份，成功后颁发会话凭证。
```

**原因**：规范描述"行为"，实现细节在 `design.md` 中。这样更换加密方式时不用改规范。

---

### 3. Delta 写成了全量

❌ **错误**（change 中的 spec.md）：
```markdown
# Auth Specification
## 完整的用户认证系统...
（复制粘贴了整个现有规范）
```

✅ **正确**：
```markdown
# Delta for Auth

## ADDED Requirements
### Requirement: 两步验证
系统必须支持 TOTP 双因素认证。

## MODIFIED Requirements
### Requirement: 会话超时
系统必须在 15 分钟无活动后过期会话。（之前：30 分钟）
```

**原因**：增量规范更清晰，归档时只合并变化部分，避免冲突。

---

### 4. 跳过规范直接写代码

❌ **错误**：propose 完直接开始写代码

✅ **正确**：
1. propose → 审阅 proposal.md
2. 生成 specs → 审阅 specs
3. 生成 design → 审阅 design
4. 生成 tasks → 审阅 tasks
5. apply 开始实现

**原因**：在 AI 理解正确之前写代码，只会得到错误的结果。

---

### 5. 归档前不检查 deltas

❌ **错误**：
```bash
/opsx:archive  # 直接归档
```

✅ **正确**：
```bash
# 先验证
openspec validate --changes

# 查看将要合并的内容
openspec show my-change --deltas-only

# 确认无误后再归档
/opsx:archive
```

**原因**：错误的 deltas 会污染主规范。

---

## 进阶常见问题

### 6. Context 写得太长

❌ **错误**：
```yaml
context: |
  （50KB 的项目文档...）
```

✅ **正确**：
```yaml
context: |
  技术栈: TypeScript + React
  详细文档见：/docs/architecture.md
```

**原因**：Context 限制 50KB，太长会影响 AI 理解。

---

### 7. Rules 用错了 Artifact ID

❌ **错误**：
```yaml
rules:
  code:  # 没有 code 这个 artifact
    - 必须有单元测试
```

✅ **正确**：
```yaml
rules:
  tasks:  # 正确的 artifact id
    - 必须包含测试任务
```

**查看可用 ID**：
```bash
openspec schemas --json
```

---

### 8. 多变更并行时不指定变更名

❌ **错误**：
```bash
/opsx:apply  # 有多个变更时不知道用哪个
```

✅ **正确**：
```bash
# 查看所有变更
openspec list

# 指定变更名
/opsx:apply add-login
```

---

## 高级常见问题

### 9. 自定义 Schema 有循环依赖

❌ **错误**：
```yaml
artifacts:
  - id: a
    requires: [b]
  - id: b
    requires: [a]  # 循环依赖！
```

✅ **正确**：
```yaml
artifacts:
  - id: proposal
    requires: []
  - id: design
    requires: [proposal]
  - id: tasks
    requires: [design]
```

**验证 Schema**：
```bash
openspec schema validate my-workflow
```

---

### 10. Shell 补全不生效

**可能原因**：
1. 没有重启终端
2. 补全脚本路径不对
3. Shell 配置没有加载

**解决方案**：
```bash
# 重新安装补全
openspec completion install bash --verbose

# 手动加载
source ~/.bashrc

# 检查是否安装成功
openspec completion which bash
```

---

## 调试技巧

### 查看指令内容

```bash
# 查看 proposal 的完整指令（包含 context 和 rules）
openspec instructions proposal --change my-change
```

### 查看 Schema 解析路径

```bash
# 查看某个 Schema 从哪里加载的
openspec schema which spec-driven

# 查看所有 Schema
openspec schema which --all
```

### 查看 JSON 输出

```bash
# 大多数命令支持 --json 输出
openspec list --json
openspec status --change my-change --json
openspec schemas --json
```
