# 06 - 多语言支持

> 让 OpenSpec 用你的母语工作

## 快速配置

在 `openspec/config.yaml` 中添加语言配置：

```yaml
context: |
  语言：中文（简体）
  所有产出物必须用简体中文撰写。
```

就这么简单！所有生成的文档都会使用中文。

---

## 支持的语言示例

### 中文（简体）

```yaml
context: |
  语言：中文（简体）
  所有产出物必须用简体中文撰写。

  技术栈: TypeScript + React
```

**效果：**
- proposal.md 用中文撰写
- specs 用中文描述需求
- tasks 用中文列出任务

### 中文（繁体）

```yaml
context: |
  語言：中文（繁體）
  所有產出物必須用繁體中文撰寫。

  技術堆疊: TypeScript + React
```

### 日文

```yaml
context: |
  言語：日本語
  すべての成果物は日本語で作成してください。

  技術スタック: TypeScript + React
```

### 韩文

```yaml
context: |
  언어: 한국어
  모든 산출물은 한국어로 작성해야 합니다.

  기술 스택: TypeScript + React
```

### 葡萄牙语

```yaml
context: |
  Idioma: Português (pt-BR)
  Todos os artefatos devem ser escritos em português brasileiro.

  Tech stack: TypeScript + React
```

## 处理技术术语

建议保留英文技术术语：

```yaml
context: |
  语言：中文（简体）
  用中文撰写，但以下内容保持英文：
  - 技术术语：API、REST、GraphQL、JWT、OAuth 等
  - 代码示例和文件路径
  - 变量名和函数名
```

**效果：**

```markdown
## 为什么

用户需要通过 OAuth 2.0 进行身份验证...

## 技术方案

使用 JWT Token 管理会话...
```

## 多语言团队

### 场景：中英混合团队

```yaml
context: |
  语言：中文（简体）
  主要内容用中文，但以下部分需要英文：
  - 文件名和目录名
  - Git commit 信息
  - API 文档

  团队成员：中国开发 + 国际合作
```

### 场景：日企项目

```yaml
context: |
  言語：日本語（主）
  English: secondary

  ドキュメントは日本語で作成してください。
  但し、コードコメントは英語で書いてください。

  技術スタック: TypeScript + React
```

## 验证语言配置

```bash
# 查看生成的指令（应该包含你的语言配置）
openspec instructions proposal --change my-change
```

**输出示例：**

```xml
<context>
语言：中文（简体）
所有产出物必须用简体中文撰写。
</context>

<template>
## Why

<!-- Explain the motivation for this change -->
...
</template>
```

AI 会看到中文语言要求，生成的文档会使用中文。

## 常见问题

### Q: 为什么有些内容还是英文？

**A:** 检查以下几点：
1. `context` 中的语言说明是否明确
2. 是否需要补充"技术术语保持英文"的说明
3. AI 模型是否支持你的语言

### Q: 能否混合使用多种语言？

**A:** 可以，在 `context` 中说明清楚即可：

```yaml
context: |
  语言规则：
  - 主要内容：中文
  - 代码注释：英文
  - Git commits：英文
  - API 文档：中英双语
```

### Q: 对 AI 模型有要求吗？

**A:** 推荐：
- 中文：Claude 3.5+、GPT-4+
- 日文：Claude 3.5+、GPT-4+
- 其他语言：建议测试后使用
