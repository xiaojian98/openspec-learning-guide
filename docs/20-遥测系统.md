# 20 - 遥测系统

> 了解 OpenSpec 如何收集匿名使用数据

## 概述

OpenSpec 收集**匿名使用统计**，帮助我们了解使用模式并改进产品。

### 收集什么？

| 数据 | 示例 | 用途 |
|------|------|------|
| 命令名称 | `init`, `list`, `archive` | 了解常用功能 |
| 版本号 | `1.2.0` | 了解用户分布 |
| 时间戳 | `2025-01-20T10:30:00Z` | 分析使用趋势 |

### 不收集什么？

- ❌ 命令参数和路径
- ❌ 文件内容
- ❌ 项目名称
- ❌ 个人身份信息（PII）

## 源码分析

### 遥测模块

```typescript
// src/telemetry/index.ts

import { PostHog } from 'posthog-node';

let client: PostHog | null = null;

// 初始化
export function initTelemetry(): void {
  const disabled = process.env.OPENSPEC_TELEMETRY === '0' ||
                   process.env.DO_NOT_TRACK === '1' ||
                   process.env.CI === 'true';

  if (disabled) {
    return;
  }

  client = new PostHog('phc_xxx', {
    host: 'https://app.posthog.com',
  });
}

// 跟踪命令
export async function trackCommand(
  commandPath: string,
  version: string
): Promise<void> {
  if (!client) return;

  client.capture({
    distinctId: getAnonymousId(),
    event: 'command_executed',
    properties: {
      command: commandPath,
      version,
      platform: process.platform,
    },
  });
}

// 关闭
export async function shutdown(): Promise<void> {
  if (client) {
    await client.shutdown();
  }
}
```

## 禁用遥测

### 方式一：环境变量

```bash
# OpenSpec 专用环境变量
export OPENSPEC_TELEMETRY=0

# 通用 DNT 标准
export DO_NOT_TRACK=1
```

### 方式二：CI 自动禁用

在 CI 环境中（`CI=true`），遥测自动禁用。

```yaml
# GitHub Actions 示例
env:
  CI: true  # 遥测自动禁用
```

## 隐私声明

### 我们承诺

1. **只收集命令名称和版本**
2. **不收集项目信息、文件内容或个人数据**
3. **数据仅用于改进产品**
4. **可以随时禁用**

### 数据保留

- 数据保留 90 天
- 之后自动删除

### 第三方服务

- 使用 [PostHog](https://posthog.com/) 进行分析
- PostHog 是开源的产品分析平台

// 1. 定义事件类型
interface TelemetryEvent {
  event: string;
  properties: Record<string, string | number | boolean>;

  event: 'schema_created',
  properties: {
    schema_type: 'custom',

  },
