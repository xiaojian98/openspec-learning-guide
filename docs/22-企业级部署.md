## 项目模板

### 企业项目模板

```
enterprise-project/
├── openspec/
│   ├── config.yaml              # 项目配置
│   ├── schemas/                 # 自定义 Schema
│   │   └── team-workflow/
│   ├── specs/                   # 规范
│   └── changes/                 # 变更
│       └── archive/
├── .claude/                     # Claude Code 配置
├── .cursor/                     # Cursor 配置
└── .github/
    ├── workflows/
    │   └── openspec-validation.yml
    ├── copilot-instructions.md
    └── PULL_REQUEST_TEMPLATE.md
```

### 项目初始化脚本

```bash
#!/bin/bash
# scripts/init-enterprise-project.sh

# 创建目录结构
mkdir -p openspec/{schemas/team-workflow/templates,specs,changes/archive}

# 复制企业配置
cp /path/to/enterprise/config.yaml openspec/
cp -r /path/to/enterprise/schemas/* openspec/schemas/

# 初始化 OpenSpec
openspec init --tools all --force

# 设置默认 Schema
openspec config schema team-workflow

echo "企业项目初始化完成！"
```

## CI/CD 集成

### GitHub Actions

```yaml
# .github/workflows/openspec-validation.yml

name: OpenSpec Validation

on:
  pull_request:
    branches: [main]
    paths:
      - 'openspec/**'

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install OpenSpec
        run: npm install -g @fission-ai/openspec@1.2.0

      - name: Validate all changes
        run: openspec validate --changes --strict

      - name: Validate all specs
        run: openspec validate --specs
```

### GitLab CI

```yaml
# .gitlab-ci.yml

openspec-validation:
  stage: test
  image: node:20
  before_script:
    - npm install -g @fission-ai/openspec@1.2.0
  script:
    - openspec validate --changes --strict
    - openspec validate --specs
  only:
    changes:
      - openspec/**/*
```

### Jenkins

```groovy
// Jenkinsfile

pipeline {
  agent any

  stages {
    stage('OpenSpec Validation') {
      when {
        changeset 'openspec/**'
      }
      steps {
        sh 'npm install -g @fission-ai/openspec@1.2.0'
        sh 'openspec validate --changes --strict'
        sh 'openspec validate --specs'
      }
    }
  }
}
```

## 安全考虑

### 敏感信息处理

```yaml
# openspec/config.yaml

context: |
  技术栈: TypeScript + React

  # ❌ 不要包含敏感信息
  # DATABASE_URL: postgresql://user:pass@host/db

  # ✅ 使用环境变量引用
  数据库: 参见环境变量 DATABASE_URL

  # ✅ 使用相对路径引用
  API 密钥配置: 参见 /docs/secrets.md
```

### 访问控制

```bash
# 使用 .gitignore 排除敏感文件
echo "openspec/config.local.yaml" >> .gitignore
echo "openspec/**/secrets/**" >> .gitignore
```

## 审计日志

### 启用审计

```yaml
# openspec/config.yaml

context: |
  审计要求:
    - 所有变更必须有审计跟踪
    - 归档时记录操作者

rules:
  proposal:
    - 必须包含变更原因
    - 必须标识变更所有者
```

### 日志格式

```

2025-01-20T14:00:00Z charlie refactor-api ARCHIVED
```
