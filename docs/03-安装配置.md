# 03 - 安装配置

> 详细的安装和配置指南

## 系统要求

| 要求 | 说明 |
|------|------|
| Node.js | ≥ 20.19.0 |
| 包管理器 | npm / pnpm / yarn / bun 任一 |
| 操作系统 | macOS / Linux / Windows |
| Git | 推荐（用于版本控制） |

### 检查 Node.js 版本

```bash
node --version
# 应该显示 v20.19.0 或更高
```

如果版本过低，请先升级 Node.js：
- 推荐使用 [nvm](https://github.com/nvm-sh/nvm) 或 [fnm](https://github.com/Schniz/fnm) 管理 Node.js 版本

## 安装方式

### 方式一：npm（推荐）

```bash
npm install -g @fission-ai/openspec@latest
```

### 方式二：pnpm

```bash
pnpm add -g @fission-ai/openspec@latest
```

### 方式三：yarn

```bash
yarn global add @fission-ai/openspec@latest
```

### 方式四：bun

```bash
bun add -g @fission-ai/openspec@latest
```

### 方式五：npx（临时使用）

```bash
npx @fission-ai/openspec@latest --help
```

### 方式六：Nix（Linux/macOS）

```bash
nix run github:Fission-AI/OpenSpec
```

## 初始化项目

### 基本初始化

```bash
cd your-project
openspec init
```

### 交互式配置

初始化时会询问：

```
? 选择你要使用的 AI 工具：
  ◉ Claude Code
  ◯ Cursor
  ◯ Windsurf
  ◯ GitHub Copilot
  ◯ Cline
  ...
```

使用 **空格** 选择/取消，**回车** 确认。

### 非交互式初始化

```bash
# 只选择 Claude Code
openspec init --tools claude

# 选择多个工具
openspec init --tools claude,cursor,windsurf

# 选择所有支持的工具
openspec init --tools all
```

## 项目配置（config.yaml）

### 基本配置

```yaml
# openspec/config.yaml
schema: spec-driven

context: |
  技术栈: TypeScript + React + Node.js
  API 规范: RESTful, JSON 响应
  测试: Vitest 单元测试, Playwright E2E
  代码风格: ESLint + Prettier

rules:
  proposal:
    - 必须包含回滚方案
    - 标识受影响的团队
  specs:
    - 使用 Given/When/Then 格式写场景
  design:
    - 复杂流程必须有流程图
```

### 配置字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `schema` | string | 默认使用的 Schema 名称 |
| `context` | string | 项目背景，会注入到所有 AI 指令中 |
| `rules` | object | 按 Artifact 类型分类的规则 |

### Context 的作用

`context` 会被注入到所有 Artifact 的 AI 指令中：

```xml
<context>
技术栈: TypeScript + React + Node.js
API 规范: RESTful, JSON 响应
...
</context>

<template>
[Schema 的模板内容]
</template>
```

这样 AI 生成的文档会自动符合你的项目规范。

### Rules 的作用

`rules` 只会注入到对应的 Artifact：

```yaml
rules:
  proposal:
    - 必须包含回滚方案    # 只在创建 proposal 时生效
  specs:
    - 使用 Given/When/Then # 只在创建 specs 时生效
```

## 全局配置

### 配置文件位置

| 平台 | 路径 |
|------|------|
| macOS/Linux | `~/.config/openspec/config.json` |
| Windows | `%APPDATA%/openspec/config.json` |

### 全局配置内容

```json
{
  "profile": "core",
  "delivery": "skills",
  "featureFlags": {}
}
```

### Profile 选项

| Profile | 包含的命令 |
|---------|-----------|
| `core` | propose, explore, apply, archive |
| `custom` | 自定义选择 |

### Delivery 选项

| Delivery | 说明 |
|----------|------|
| `skills` | 使用 `.claude/skills/` 目录 |
| `commands` | 使用 `.claude/commands/` 目录 |
| `both` | 同时生成两种 |

## 常见问题

### Q: 安装后提示找不到命令？

**A:** 确保 npm 全局安装路径在 PATH 中：

```bash
# 检查 npm 全局路径
npm config get prefix

# 添加到 PATH（bash）
export PATH="$(npm config get prefix)/bin:$PATH"
```

### Q: 初始化时没有询问工具选择？

**A:** 可能之前已经初始化过。使用 `--force` 强制重新初始化：

```bash
openspec init --force
```

### Q: Windows 上路径有问题？

**A:** OpenSpec 是跨平台的，使用正确的路径分隔符。如有问题，请在 [GitHub Issues](https://github.com/Fission-AI/OpenSpec/issues) 反馈。
