## 核心模块

### 1. CLI 层（src/cli/）

```
src/cli/
└── index.ts              # 命令注册和启动
```

**职责**：
- 使用 Commander.js 注册所有命令
- 处理全局选项（--no-color 等）
- 调用遥测系统

**关键代码**：
```typescript
import { Command } from 'commander';

const program = new Command();
program
  .name('openspec')
  .description('AI-native system for spec-driven development')
  .version(version);

// 注册命令
program.command('init [path]').action(initCommand);
program.command('list').action(listCommand);
// ...

program.parse();
```

## 数据流

### 初始化流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    openspec init 流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   CLI                                                          │
│   ┌──────────┐                                                 │
│   │ parse    │                                                 │
│   │ args     │                                                 │
│   └────┬─────┘                                                 │
│        │                                                        │
│        ▼                                                        │
│   InitCommand                                                  │
│   ┌──────────────────────────────────────────────────────────┐ │
│   │ 1. 检测当前目录状态                                       │ │
│   │ 2. 查询全局配置（profile, delivery）                     │ │
│   │ 3. 交互式选择 AI 工具                                     │ │
│   │ 4. 生成 .claude/ 目录结构                                 │ │
│   │ 5. 创建 openspec/ 目录结构                                │ │
│   │ 6. 生成 config.yaml（可选）                               │ │
│   └──────────────────────────────────────────────────────────┘ │
│        │                                                        │
│        ▼                                                        │
│   CommandGeneration                                            │
│   ┌──────────────────────────────────────────────────────────┐ │
│   │ 1. 根据选中的工具获取适配器                               │ │
│   │ 2. 读取 workflow 模板                                     │ │
│   │ 3. 应用 context 和 rules                                  │ │
│   │ 4. 生成 .claude/commands/ 和 .claude/skills/             │ │
│   └──────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Status 流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    openspec status 流程                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   CLI                                                          │
│   ┌──────────┐                                                 │
│   │ parse    │                                                 │
│   │ args     │                                                 │
│   └────┬─────┘                                                 │
│        │                                                        │
│        ▼                                                        │
│   StatusCommand                                                │
│   ┌──────────────────────────────────────────────────────────┐ │
│   │ 1. 加载 Schema（从项目或全局）                            │ │
│   │ 2. 创建 ArtifactGraph                                     │ │
│   │ 3. 检查文件系统状态                                       │ │
│   │ 4. 计算每个 artifact 的状态                               │ │
│   │ 5. 输出结果（JSON 或文本）                                │ │
│   └──────────────────────────────────────────────────────────┘ │
│        │                                                        │
│        ▼                                                        │
│   ArtifactGraph                                                │
│   ┌──────────────────────────────────────────────────────────┐ │
│   │ getBuildOrder()     → 拓扑排序                           │ │
│   │ getNextArtifacts()  → 下一个可创建的                     │ │
│   │ getBlocked()        → 被阻塞的                           │ │
│   │ isComplete()        → 是否完成                           │ │
│   └──────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
