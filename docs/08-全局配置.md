# 08 - 全局配置

> 跨项目共享配置和 Schema

## 配置文件位置

| 平台 | 路径 |
|------|------|
| macOS/Linux | `~/.config/openspec/` |
| Windows | `%APPDATA%/openspec/` |

### 配置文件结构

```
~/.config/openspec/
├── config.json        # 全局配置
└── schemas/           # 全局 Schema（可选）
    └── my-workflow/
        ├── schema.yaml
        └── templates/
```

---

## 全局配置（config.json）

```json
{
  "profile": "core",
  "delivery": "both",
  "workflows": [],
  "featureFlags": {}
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `profile` | string | `core` 或 `custom` |
| `delivery` | string | `skills`、`commands` 或 `both` |
| `workflows` | string[] | custom profile 使用的 workflow 列表 |
| `featureFlags` | object | 实验性功能开关 |

---

## 全局 Schema

### 为什么用全局 Schema？

- 跨项目共享
- 版本控制（不跟随单个项目）
- 团队统一标准

### 创建全局 Schema

```bash
# 创建目录
mkdir -p ~/.local/share/openspec/schemas/enterprise

# 复制 Schema
cp -r openspec/schemas/spec-driven/* ~/.local/share/openspec/schemas/enterprise/

# 编辑
vim ~/.local/share/openspec/schemas/enterprise/schema.yaml
```

### Schema 解析顺序

```
1. CLI 参数 --schema <path>
2. 项目目录 openspec/schemas/<name>/
3. 全局目录 ~/.local/share/openspec/schemas/<name>/
4. 内置 Schema
```

---

## 管理全局配置

### CLI 命令

```bash
# 查看全局配置
openspec config list

# 设置 profile
openspec config profile custom

# 设置 delivery
openspec config delivery skills

# 选择 workflows（custom profile）
openspec config workflows
```
