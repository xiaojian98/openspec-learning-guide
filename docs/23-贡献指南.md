# 23 - 贡献指南

> 如何为 OpenSpec 做贡献

## 贡献方式

### 1. 报告 Bug

```markdown
## Bug 描述

**OpenSpec 版本**: 1.2.0
**操作系统**: macOS 14.0
**Node.js 版本**: 20.10.0

### 复现步骤

1. 运行 `openspec init`
2. 选择 Claude Code
3. 报错：...

### 期望行为

应该成功初始化。

### 实际行为

报错：...

### 日志

```
（粘贴错误日志）
```
```

提交到：https://github.com/Fission-AI/OpenSpec/issues

---

### 2. 提交功能建议

```markdown
## 功能建议

### 问题描述

当前 OpenSpec 不支持...

### 建议方案

添加一个新命令 `openspec diff`...

### 替代方案

也可以通过...

### 附加信息

类似工具的参考...
```

---

### 3. 提交代码

#### 小改动

```bash
# 1. Fork 仓库
gh repo fork Fission-AI/OpenSpec --clone

# 2. 创建分支
git checkout -b fix/my-fix

# 3. 修改代码
# ...

# 4. 运行测试
pnpm test

# 5. 提交
git commit -m "fix(scope): 描述修复内容"

# 6. 推送并创建 PR
git push origin fix/my-fix
gh pr create
```

#### 大改动

**先创建 OpenSpec 变更提案！**

```bash
# 1. 在 OpenSpec 仓库中创建变更
/opsx:propose 添加 diff 命令

# 2. 等待维护者审阅

# 3. 批准后再实现
/opsx:apply

# 4. 完成后归档
/opsx:archive
```

## 开发环境设置

### 系统要求

- Node.js 20.19.0+
- pnpm 9+
- Git

### 安装步骤

```bash
# 克隆仓库
git clone https://github.com/Fission-AI/OpenSpec.git
cd OpenSpec

# 安装依赖
pnpm install

# 构建
pnpm run build

# 运行测试
pnpm test

# 本地开发
pnpm run dev:cli
```

## 代码规范

### 提交消息

使用 Conventional Commits：

```
<type>(<scope>): <subject>

类型：
- feat: 新功能
- fix: Bug 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

示例：
- feat(cli): add diff command
- fix(archive): handle missing spec files
- docs(readme): update installation steps
```

### 代码风格

- ESLint + Prettier
- 严格 TypeScript
- 单元测试覆盖

```bash
# 检查代码
pnpm lint

# 类型检查
pnpm exec tsc --noEmit
```

## 添加新 AI 工具适配器

### 步骤

```typescript
// 1. 创建适配器文件
// src/core/command-generation/adapters/my-tool.ts

import type { ToolCommandAdapter, CommandContent } from '../types.js';

export const myToolAdapter: ToolCommandAdapter = {
  toolId: 'my-tool',

  getFilePath(commandId: string): string {
    return `.my-tool/commands/${commandId}.md`;
  },

  formatFile(content: CommandContent): string {
    return `# ${content.name}\n\n${content.body}`;
  }
};

// 2. 注册适配器
// src/core/command-generation/adapters/index.ts

import { myToolAdapter } from './my-tool.js';

export const adapters = {
  'my-tool': myToolAdapter,
};

// 3. 添加到工具列表
// src/core/config.ts

export const AI_TOOLS: AIToolOption[] = [
  // ...
  {
    value: 'my-tool',
    name: 'My Tool',
    skillsDir: '.my-tool/skills',
    commandsDir: '.my-tool/commands',
  },
];

// 4. 添加测试
// test/core/command-generation/adapters/my-tool.test.ts

describe('myToolAdapter', () => {
  it('should generate correct file path', () => {
    expect(myToolAdapter.getFilePath('explore'))
      .toBe('.my-tool/commands/explore.md');
  });
});
```

import { Command } from 'commander';

export function registerMyCommand(program: Command): void {
  program
    .command('my-command [arg]')
    .description('Do something')
    .option('--option', 'Some option')
    .action(async (arg, options) => {
      // 实现逻辑
    });

// 2. 在 CLI 入口注册
// src/cli/index.ts

import { registerMyCommand } from '../commands/my-command.js';

// ...

registerMyCommand(program);

// 3. 添加补全支持
// src/core/completions/command-registry.ts

this.registerCommand({
  name: 'my-command',

  options: [
    { name: 'option', description: 'Some option' },

// test/commands/my-command.test.ts

describe('my-command', () => {
  it('should do something', async () => {
    // 测试逻辑
  });
});

## 发布流程

### 自动发布

1. 合并 PR 到 `main`
2. Changeset 自动创建版本 PR
3. 维护者合并版本 PR
4. 自动发布到 npm

### 手动发布（仅维护者）

```bash
# 更新版本
pnpm changeset version

# 发布
pnpm run release:ci
```
