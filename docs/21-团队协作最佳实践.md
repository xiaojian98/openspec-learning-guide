## 团队配置

### 共享 config.yaml

```yaml
# openspec/config.yaml

schema: team-enterprise

context: |
  团队：后端开发组
  项目：用户服务

  技术栈:
    - Go 1.21
    - PostgreSQL 15
    - Redis 7

  代码规范:
    - 使用 golangci-lint
    - 单元测试覆盖率 > 80%
    - 所有 API 必须有文档

  审查要求:
    - 至少 1 人审查
    - 安全相关变更需要安全组审查

rules:
  proposal:
    - 必须说明影响的服务和依赖
    - 必须包含回滚方案
    - 必须评估性能影响
  design:
    - 必须包含数据库变更（如有）
    - 必须包含 API 变更（如有）
    - 必须说明部署顺序
  tasks:
    - 每个任务关联 Issue
    - 测试任务必须明确
```

### 统一 Schema

```yaml
# openspec/schemas/team-enterprise/schema.yaml

name: team-enterprise
version: 1
description: 团队企业级工作流

artifacts:
  - id: proposal
    generates: proposal.md
    template: proposal.md
    requires: []

  - id: security-check
    generates: security-check.md
    template: security-check.md
    instruction: |
      评估安全影响：
      - 认证授权
      - 数据安全
      - API 安全
    requires: [proposal]

  - id: specs
    generates: specs/**/*.md
    template: spec.md
    requires: [proposal, security-check]

  - id: design
    generates: design.md
    template: design.md
    requires: [specs]

  - id: tasks
    generates: tasks.md
    template: tasks.md
    requires: [design]

apply:
  requires: [tasks]
  tracks: tasks.md
```

## 命名规范

### 变更命名

```
<类型>-<描述>

类型：
- feature: 新功能
- fix: Bug 修复
- refactor: 重构
- docs: 文档
- infra: 基础设施

示例：
- feature-add-user-auth
- fix-login-timeout
- refactor-auth-module
- docs-api-reference
- infra-docker-config
```

### 分支命名

```bash
# 与变更名保持一致
git checkout -b feature/add-user-auth
git checkout -b fix/login-timeout
```

## Git 工作流

### 变更与 Git 集成

```bash
# 1. 创建新分支
git checkout -b feature/add-dark-mode

# 2. 创建 OpenSpec 变更
/opsx:propose 添加深色模式

# 3. 实现并提交
git add openspec/
git commit -m "feat: add dark mode proposal"

# 4. 继续实现
/opsx:apply

git add .
git commit -m "feat: implement dark mode"

# 5. 创建 PR
gh pr create --title "feat: 添加深色模式" --body-file openspec/changes/add-dark-mode/proposal.md

# 6. 合并后归档
/opsx:archive
```

## 代码审查

### PR 模板

```markdown
## 变更说明

<!-- 从 proposal.md 复制关键内容 -->

## OpenSpec 变更

- [ ] proposal.md 已审阅
- [ ] specs/ 已审阅
- [ ] design.md 已审阅
- [ ] tasks.md 任务已完成

## 测试

- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 手动测试通过

## 检查清单

- [ ] 代码符合团队规范
- [ ] 没有引入新的安全风险
- [ ] 文档已更新
```

### 审查要点

| 审查项 | 检查内容 |
|--------|---------|
| Proposal | 意图清晰、范围合理 |
| Specs | 需求完整、场景覆盖 |
| Design | 技术方案可行 |
| Tasks | 任务粒度合适 |

## 多人协作

### 避免冲突

1. **每个变更有独立目录**
2. **Delta Specs 只描述变化**
3. **归档时才合并到主 specs**

### 并行开发

```
openspec/changes/
├── alice/
│   └── feature-add-api/
├── bob/
│   └── fix-login-bug/
└── shared/
    └── refactor-auth/
```

### 同步机制

```bash
# 拉取最新变更
git pull

# 更新 OpenSpec
openspec update

# 查看状态
openspec list
```

## CI/CD 集成

### GitHub Actions

```yaml
# .github/workflows/openspec-validation.yml

name: OpenSpec Validation

on:
  pull_request:
    paths:
      - 'openspec/**'

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install OpenSpec
        run: npm install -g @fission-ai/openspec@latest

      - name: Validate changes
        run: openspec validate --changes --strict
```
