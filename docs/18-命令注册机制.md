# 18 - 命令注册机制

> 了解 OpenSpec 如何使用 Commander.js 注册命令

## Commander.js 简介

OpenSpec 使用 [Commander.js](https://github.com/tj/commander.js) 作为 CLI 框架。

### 基本用法

```typescript
import { Command } from 'commander';

const program = new Command();

program
  .name('openspec')
  .description('AI-native system for spec-driven development')
  .version('1.2.0');

program.parse();
```

## 命令类型

### 1. 简单命令

```typescript
program
  .command('list')
  .description('List items (changes by default)')
  .option('--specs', 'List specs instead of changes')
  .option('--json', 'Output as JSON')
  .action(async (options) => {
    const listCommand = new ListCommand();
    await listCommand.execute('.', options.specs ? 'specs' : 'changes', {
      json: options.json,
    });
  });
```

### 2. 带参数的命令

```typescript
program
  .command('show [item-name]')
  .description('Show a change or spec')
  .option('--json', 'Output as JSON')
  .action(async (itemName, options) => {
    const showCommand = new ShowCommand();
    await showCommand.execute(itemName, options);
  });
```

### 3. 嵌套命令

```typescript
// 父命令
const changeCmd = program
  .command('change')
  .description('Manage OpenSpec changes');

// 子命令
changeCmd
  .command('show [change-name]')
  .description('Show a change proposal')
  .action(async (changeName, options) => {
    // ...
  });

changeCmd
  .command('list')
  .description('List all active changes')
  .action(async (options) => {
    // ...
  });
```

### 4. 隐藏命令

```typescript
program
  .command('__complete <type>', { hidden: true })
  .description('Output completion data (internal use)')
  .action(async (type) => {
    // ...
  });
```

## 生命周期钩子

### preAction 钩子

```typescript
program.hook('preAction', async (thisCommand, actionCommand) => {
  // 处理全局选项
  const opts = thisCommand.opts();
  if (opts.color === false) {
    process.env.NO_COLOR = '1';
  }

  // 显示遥测通知
  await maybeShowTelemetryNotice();

  // 记录命令执行
  const commandPath = getCommandPath(actionCommand);
  await trackCommand(commandPath, version);
});
```

### postAction 钩子

```typescript
program.hook('postAction', async () => {
  // 关闭遥测
  await shutdown();
});
```

## 命令路径获取

```typescript
function getCommandPath(command: Command): string {
  const names: string[] = [];
  let current: Command | null = command;

  while (current) {
    const name = current.name();
    if (name && name !== 'openspec') {
      names.unshift(name);
    }
    current = current.parent;
  }

  return names.join(':') || 'openspec';
}

// 示例输出：
// 'init'
// 'change:show'
// 'completion:install'
```

## 选项处理

### 必填选项

```typescript
program
  .command('archive [change-name]')
  .option('-y, --yes', 'Skip confirmation prompts')
  .option('--skip-specs', 'Skip spec update operations')
  .action(async (changeName, options) => {
    // options.yes
    // options.skipSpecs
  });
```

### 可变参数

```typescript
program
  .command('feedback <message>')
  .option('--body <text>', 'Detailed description')
  .action(async (message, options) => {
    // message 是必填参数
    // options.body 是可选参数
  });
```

### 允许未知选项

```typescript
program
  .command('show [item-name]')
  .allowUnknownOption(true)
  .action(async (itemName, options) => {
    // 可以处理未定义的选项
  });
```

## 错误处理

### 统一错误处理

```typescript
.action(async (...args) => {
  try {
    // 命令逻辑
  } catch (error) {
    console.log(); // 空行
    ora().fail(`Error: ${(error as Error).message}`);
    process.exit(1);
  }
});
```

### 退出码

```typescript
// 使用 process.exitCode 而不是 process.exit()
// 这样可以正确执行清理操作
process.exitCode = 1;

// 或者在需要立即退出时
process.exit(1);
```
